# План создания Android-приложения для клуба «Гармония»

Поскольку текущий проект написан на **React (Next.js 15)**, у нас есть огромное преимущество — мы можем переиспользовать существующий код, дизайн-систему и базу данных. 

Для создания мобильного приложения для родителей предлагаю следующий стратегический план:

---

## 1. Выбор технологического стека

Вместо разработки "с нуля" на Java/Kotlin (что дорого и долго), нам подходят два современных подхода:

### Вариант А: React Native + Expo (Рекомендуемый 🚀)
*   **Почему:** Полноценное нативное мобильное приложение. Нативнее анимации, глубокая интеграция с функциями телефона (Push-уведомления, камера, календарь).
*   **Плюс:** Те же технологии, что и на сайте (React, TypeScript, Tailwind).
*   **Минус:** Потребуется написать интерфейс заново (используя компоненты React Native вместо HTML).

### Вариант Б: Next.js + Capacitor (Самый быстрый ⚡)
*   **Почему:** Capacitor "оборачивает" ваш текущий мобильный сайт в Android-приложение, которое можно выложить в Google Play.
*   **Плюс:** Разработка займет всего пару дней. Единая кодовая база с сайтом.
*   **Минус:** Чуть медленнее нативных приложений, сложнее делать сложные мобильные анимации.

---

## 2. Адаптация Backend (API)
Текущий проект активно использует **Next.js Server Actions**, которые жестко привязаны к сайту. Мобильному приложению потребуется классический API.
*   **Что сделать:** Продублировать основные функции корзины, записи и расписания в Next.js Route Handlers (`src/app/api/...`), чтобы мобильное приложение могло общаться с вашей базой Prisma (PostgreSQL).
*   **Авторизация:** Внедрить JWT-токены или перевести авторизацию на более универсальное решение (например, NextAuth / Auth.js с поддержкой мобильных клиентов).

---

## 3. Ключевой функционал приложения (MVP)

Мобильное приложение должно решать задачи **удержания (Retention)** и давать родителям удобный доступ к информации о ребенке, в то время как сайт служит для привлечения новых клиентов.

1. **Личный кабинет родителя:**
   *   Профиль ребенка, текущий уровень (Starters, Movers), прогресс.
   *   Остаток занятий на абонементе.
2. **Расписание и Уведомления (Киллер-фича):**
   *   Календарь занятий с возможностью синхронизации с Google Календарем.
   *   **Push-уведомления:** Напоминания за 2 часа до урока, уведомления о пересадках, об окончании абонемента.
3. **Оплата абонементов:**
   *   Встроенная оплата (например, рекуррентные платежи или СБП).
4. **Обратная связь (Связь с преподавателем):**
   *   Короткие отчеты после уроков ("Сегодня Петя отлично справился с тестом!").
   *   Чат с администратором клуба.

---

## 4. Дорожная карта (Roadmap) разработки

*   **Этап 1: Проектирование API и Авторизации (1-2 недели)**
    *   Создание REST API маршрутов на Next.js.
    *   Реализация безопасной мобильной авторизации.
*   **Этап 2: Верстка и навигация (Рекомендуется Expo) (2-3 недели)**
    *   Настройка базовой навигации (Bottom Tabs).
    *   Перенос дизайн-токенов Tailwind на мобильные устройства.
    *   Создание экранов: Главная, Расписание, Профиль.
*   **Этап 3: Интеграция с базой и логика (2 недели)**
    *   Подключение приложения к удаленной БД через API.
    *   Настройка уведомлений (Expo Push Notifications).
*   **Этап 4: Тестирование и Публикация (1-2 недели)**
    *   Тестирование на реальных Android-устройствах.
    *   Сборка `.aab` файла.
    *   Регистрация в Google Play Console и публикация (проверка Google занимает 3-7 дней).

---

## Вывод
Для языкового клуба мобильное приложение — это инструмент лояльности. Если нужен быстрый старт "для галочки", то мы можем использовать Capacitor и упаковать сайт за пару дней. 

Но если вы хотите дать родителям крутой опыт, Push-уведомления об уроках и личный кабинет, то **лучше собирать отдельное приложение на React Native (Expo)**, переиспользуя логику вашего текущего Next.js бэкенда.
