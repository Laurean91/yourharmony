# Попап занятия в расписании

**Дата:** 2026-04-03  
**Файл реализации:** `src/components/WeekSchedule.tsx`

---

## Цель

Заменить боковую панель справа (`md:w-72`) на всплывающий попап, который появляется над кликнутым блоком занятия. На мобильных устройствах — центрированный модал с backdrop.

---

## Поведение

### Десктоп (≥ 768px)

- Клик на блок занятия → попап появляется **над блоком** со стрелкой-хвостиком (↓), указывающей на блок
- Если над блоком недостаточно места (занятие в верхней части сетки) — попап появляется **снизу** с перевёрнутой стрелкой (↑)
- Ширина попапа: **260px**, фиксированная
- Позиционирование: абсолютное через `getBoundingClientRect()` + `createPortal` в `document.body` — чтобы не обрезался родительским `overflow: auto`
- Клик вне попапа (на backdrop или фон) → закрывает попап

### Мобильный (< 768px)

- Тот же попап появляется **по центру экрана** как модальное окно
- Полупрозрачный backdrop (`rgba(0,0,0,0.35)`) перекрывает всё
- Попап занимает ширину `calc(100vw - 32px)`, максимум `320px`

## Содержимое попапа

### Состояние: просмотр

1. **Шапка:** дата + время (`Ср, 7 января · 10:00`), badge тега (`Индивидуальное` / `Группа`), кнопка закрытия `×`
2. **Посещаемость** (если есть ученики): метка «Посещаемость N/N», список учеников — клик по строке отмечает/снимает посещаемость (поведение без изменений)
3. **Если учеников нет:** текст «Ученики не назначены»
4. **Кнопки:** «Редактировать» + «Удалить»

### Состояние: редактирование

- Клик «Редактировать» → форма редактирования раскрывается **внутри попапа** (под кнопками, через `border-top`)
- Поля: дата, время, тип, тема, ученики (с поиском), заметки — идентично текущей реализации
- Если содержимое не вмещается — попап скроллируется (`overflow-y: auto`, `max-height: 80vh`)
- Кнопки «Сохранить» / «Отмена» в конце формы

## Архитектура

### Новый компонент: `LessonPopover`

Выносится в отдельный компонент (или остаётся inline — по выбору при реализации). Получает пропсы:

```ts
interface LessonPopoverProps {
  lesson: Lesson
  students: Student[]
  anchorRect: DOMRect        // getBoundingClientRect() кликнутого блока
  onClose: () => void
  onDelete: (id: string) => void
  onAttendance: (lessonId: string, studentId: string, attended: boolean) => void
  onEdit: (id: string, formData: FormData) => Promise<void>
}
```

Рендерится через `ReactDOM.createPortal(…, document.body)`.

### Изменения в `WeekSchedule.tsx`

- **Удалить:** блок `{(selected || addSlot) && (<div className="w-full md:w-72 ...">)}` (боковая панель)
- **Удалить:** `flex flex-col md:flex-row` обёртку — таймлайн занимает всю ширину
- **Добавить:** state `anchorRect: DOMRect | null` — запоминается при клике на занятие
- **Изменить:** `onClick` блока занятия — сохраняет `e.currentTarget.getBoundingClientRect()` в state
- **Добавить:** рендер `<LessonPopover>` когда `selected !== null`
- **Форма добавления занятия** (`addSlot`) — аналогично превращается в попап (тот же компонент с другим режимом, или отдельный маленький попап)

### Позиционирование попапа

```ts
// Определяем: вверх или вниз
const spaceAbove = anchorRect.top
const spaceBelow = window.innerHeight - anchorRect.bottom
const showAbove = spaceAbove > 300 || spaceAbove > spaceBelow

// Координаты
const top = showAbove
  ? anchorRect.top + window.scrollY - popoverHeight - 10
  : anchorRect.bottom + window.scrollY + 10

const left = Math.max(8, Math.min(
  anchorRect.left + anchorRect.width / 2 - 130, // центрирование
  window.innerWidth - 268                         // не выходить за правый край
))
```

### Мобильный режим

Определяется хуком или `window.innerWidth < 768` в момент клика. На мобильном `top/left` игнорируются, вместо этого модал центрируется через CSS (`position:fixed; inset:0; display:flex; align-items:center; justify-content:center`).

## Что не меняется

- Логика посещаемости, редактирования, удаления — без изменений
- Drag-and-drop перенос занятий — без изменений
- Форма добавления нового занятия (`addSlot`) — остаётся, можно оставить как попап в том же стиле
- Все Server Actions (`updateLesson`, `deleteLesson`, `markAttendance`) — без изменений
